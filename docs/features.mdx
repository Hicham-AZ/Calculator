# Features

The Calculator application offers a comprehensive set of features for both basic and advanced mathematical operations.

## Calculator Modes

### Standard Mode

The standard calculator mode provides all essential arithmetic operations needed for everyday calculations.

#### Available Operations
- **Addition (+)**: Add two or more numbers
- **Subtraction (-)**: Subtract numbers, supports negative numbers
- **Multiplication (*)**: Multiply numbers with decimal precision
- **Division (/)**: Divide numbers with proper error handling for division by zero
- **Modulus (%)**: Calculate remainder of division operations
- **Decimal Point (.)**: Support for decimal numbers
- **Clear (C)**: Reset calculator to initial state
- **Backspace (←)**: Remove the last entered digit

#### Features
- **Number Input**: Supports both integers and decimal numbers
- **Continuous Operations**: Chain multiple operations together
- **Input Validation**: Prevents invalid input using regex patterns
- **Result Display**: Automatic formatting of results (removes trailing zeros for integers)

### Scientific Mode

Scientific mode extends the calculator with advanced mathematical functions.

#### Additional Operations
- **Square Root (√)**: Calculate square root of a number
- **Power (pow)**: Raise numbers to any power (x^y)
- **Natural Logarithm (ln)**: Calculate natural logarithm of a number

#### Mode Switching
- Switch between Standard and Scientific modes via dropdown
- Dynamic UI resizing (Standard: 410x600px, Scientific: 490x600px)
- Scientific buttons are hidden in Standard mode for cleaner interface

## Input Validation

### Number Format Validation
The calculator uses sophisticated regex patterns for input validation:

```java
// Regex pattern for valid numbers
"([-]?\\d+[.]\\d*)|(\\d+)|(-\\d+)"
```

This pattern supports:
- **Positive integers**: `123`
- **Negative integers**: `-456`
- **Positive decimals**: `12.34`
- **Negative decimals**: `-56.78`
- **Leading zeros**: Automatically handled

### Error Prevention
- **Invalid Input Rejection**: Non-numeric input is ignored
- **Decimal Point Control**: Only one decimal point per number
- **Operator Chaining**: Prevents invalid operator sequences
- **Division by Zero**: Handled gracefully (returns infinity)

## Limitations and Considerations

### Current Limitations
- **Single Window**: One calculator instance at a time
- **No History**: Previous calculations are not stored
- **Limited Scientific Functions**: Basic scientific operations only
- **No Memory Functions**: M+, M-, MR, MC not implemented

### Future Enhancement Opportunities
- **Calculation History**: Store and recall previous calculations
- **Memory Functions**: Add standard calculator memory operations  
- **More Scientific Functions**: Trigonometric functions, constants
- **Keyboard Input**: Support for keyboard number entry
- **Copy/Paste**: Clipboard integration for results

## Usage Tips

### Best Practices
1. **Clear Before New Calculations**: Use 'C' to start fresh calculations
2. **Check Display**: Verify numbers before pressing operators
3. **Use Backspace**: Correct single digit mistakes with '←'
4. **Chain Operations**: Take advantage of continuous calculation support
5. **Theme Selection**: Choose theme based on lighting conditions

- **Basic Arithmetic**: Everyday calculations like tips, taxes, measurements
- **Scientific Calculations**: Square roots, powers, logarithms for STEM work
- **Financial Calculations**: Percentages, interest calculations
- **Engineering**: Quick calculations during technical work
